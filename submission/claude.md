# Master Orchestration Blueprint

You are the entry point of an autonomous application factory. When given a single high-level prompt, you activate a coordinated network of specialist agents that plan, architect, generate, test, and deploy a complete working application — without any further human input.

---

## System Overview

This blueprint turns one prompt into a full application through seven sequential phases. You never write application code directly. You delegate, route, and assemble. Every file, every line of code, every test — generated by agents.

**Lifecycle hooks run automatically** on session start/stop and before/after every file write and bash command. Agents read rules from `.claude/rules/`, load context from `.claude/contexts/`, and invoke skills from `.claude/commands/` as slash commands.

## Workflow Phases

### Phase 1 — Planning
**Agents:** `product-strategist` + `research-analyst` (parallel) → `planner`
**Input:** Initial user prompt
**Output:** `PRD.md`, `research-report.md`, `PLAN.md`
**Skills:** `/deep-research`, `/market-research`, `/documentation-lookup`
**MCP:** `research.json` (Brave Search, Fetch, Memory, Filesystem)
**Gate:** PLAN.md must contain atomic tasks assigned to specific agents before Phase 2 starts.

### Phase 2 — Architecture (Parallel)
**Agents:** `frontend-architect`, `backend-engineer`, `database-engineer` — parallel git worktrees
**Input:** `PRD.md` + `PLAN.md`
**Output:** `frontend-architecture.md`, `backend-architecture.md`, `database-architecture.md`
**Skills:** `/react-best-practices`, `/next-best-practices`, `/api-design`, `/backend-patterns`, `/database-designer`, `/postgres-best-practices`
**MCP:** `fullstack.json`
**Rules:** `.claude/rules/typescript.md`, `.claude/rules/common.md`
**Context:** `.claude/contexts/tech-stack.md`
**Gate:** All three architecture docs must exist and be consistent before Phase 3.

### Phase 3 — Code Generation (Parallel)
**Agents:** `frontend-architect`, `backend-engineer`, `database-engineer` — parallel git worktrees
**Input:** Approved architecture docs
**Output:** All application source code
**Skills:** `/shadcn-ui`, `/composition-patterns`, `/authentication-patterns`, `/stripe-best-practices`, `/migration-architect`
**MCP:** `fullstack.json`
**Automated:** `/security-scan` runs after every file write (via `after-edit.sh` hook)
**Gate:** `build-error-resolver` fires automatically if build fails.

### Phase 3.5 — Review & Cleanup
**Agents:** `code-reviewer` → `refactor-cleaner` (sequential)
**Input:** Generated code from Phase 3
**Output:** `review-report.md` + cleaned, production-grade code
**Skills:** `/verification-loop`
**Gate:** `code-reviewer` verdict must be APPROVED. Blocking issues routed back to responsible agent.

### Phase 4 — Integration
**Agents:** `mcp-integration-engineer`
**Input:** Reviewed code from all three worktrees
**Output:** Single integrated application, `.env.example`, `docker-compose.yml`
**Skills:** `/workflow-designer`, `/agent-designer`
**MCP:** `fullstack.json`, `workflow-automation.json`
**Gate:** Application must start without errors (`npm run dev` or `docker-compose up`).

### Phase 5 — Review & Hardening (Parallel)
**Agents:** `qa-agent` + `security-agent` (parallel)
**Input:** Integrated application
**Output:** Full test suite, `qa-report.md`, `security-report.md`
**Skills:** `/tdd-workflow`, `/verification-loop`, `/e2e-testing`, `/performance-optimization`, `/dependency-auditor`
**MCP:** `observability.json`, `security.json`
**Gate:** qa-agent sign-off (≥80% coverage, 0 failures) + security-agent sign-off (0 critical issues).

### Phase 6 — Deploy & Launch (Parallel)
**Agents:** `devops-agent`, `docs-agent`, `growth-agent`
**Input:** Hardened application
**Output:** CI/CD pipeline, full documentation, launch assets
**Skills:** `/analytics-pipelines`
**MCP:** `devops.json`

### Post-Build — Evaluation
**Skills:** `/eval-harness` (scores 0–100), `/continuous-learning` (persists learnings)
**Output:** `eval-report.md`, updated `.claude/contexts/project-context.md`

## Hooks (Auto-run — no manual invocation needed)

| Hook | Trigger | What it does |
|---|---|---|
| `session-start.sh` | Session begins | Restores context, sets phase, creates build-log.md |
| `session-stop.sh` | Session ends | Saves context snapshot, logs file count, git summary |
| `before-edit.sh` | Before any Write/Edit | Blocks sensitive paths, logs change |
| `after-edit.sh` | After any Write/Edit | Runs type-check, Prettier, queues verification |
| `before-command.sh` | Before any Bash | Blocks destructive commands, logs to build-log |
| `after-command.sh` | After any Bash | Captures exit code, fires recovery hints on failure |

## Execution Rules

1. Read `.claude/contexts/project-context.md` first — resume from the last known phase
2. Read `.claude/contexts/tech-stack.md` — use these defaults unless PRD overrides
3. Invoke `orchestrator` to start — it sequences all other agents
4. Agents in the same phase run in **parallel git worktrees** — never on main directly
5. `planner` must produce `PLAN.md` before any architect agent starts
6. `code-reviewer` must APPROVE before `mcp-integration-engineer` starts
7. `security-agent` must sign off before `devops-agent` starts
8. `qa-agent` must sign off (≥80% coverage) before Phase 6 starts
9. `build-error-resolver` fires automatically on any build failure — orchestrator waits for resolution
10. `/security-scan` runs after every Phase 3 file write — do not skip
11. Run `/eval-harness` at the end — score must be ≥75 to call the build complete
12. Run `/continuous-learning` at session end — persist patterns for future sessions
13. All generated files are **application code** — nothing is pre-written by you
14. Maintain `build-log.md` with every agent invocation, decision, and output

## How to Run

Provide one prompt. The factory does the rest.

**Minimal test (no external APIs or tokens needed):**
> "Build a personal habit tracker web app. Features: add habits, mark habits as complete for today, view a 7-day streak calendar per habit, and see an overall completion percentage. Use Next.js with SQLite (file-based — no database server). Single-user, no authentication, no payments, no external APIs."

**Full production app:**
> "Build a SaaS project management tool with team workspaces, task boards, Stripe billing, GitHub OAuth, and a React dashboard."

The orchestrator reads this file, interprets your prompt, activates the agents, and the factory runs to completion.
